歡迎來到資料庫基礎篇。資料是應用程式的心臟。這集我們將探討關聯式資料庫的基礎結構、核心操作指令以及基礎維運概念。無論你使用哪種 SQL 資料庫，這些都是共通的語言。

我們從關聯式資料庫的核心標準 SQL 開始。

SQL
SQL 是 Structured Query Language 的縮寫
SQL is the standard language for relational database management.
SQL 是關聯式資料庫管理的標準語言。

在儲存資料前，我們需要定義資料的骨架，也就是 Schema 與相關結構。

綱要
Schema
The database schema defines the structure of tables and relationships.
資料庫綱要定義了資料表與關聯的結構。

資料表
Table
Create a new table to store user information.
建立一個新資料表來儲存使用者資訊。

欄位
Column
Add a new column for the email address in the user table.
在使用者資料表中新增一個 email 欄位。

列
Row
Insert a new row into the table for every new registration.
每當有新註冊時，插入一列新資料到資料表中。

主鍵
Primary Key
The ID column is the primary key and must be unique.
ID 欄位是主鍵，且必須是唯一的。

外鍵
Foreign Key
Create a foreign key to link orders to the customer table.
建立一個外鍵將訂單連結到客戶資料表。

索引
Index
Add an index on the username column to improve query speed.
在使用者名稱欄位加入索引以改善查詢速度。

有了結構後，接著就是對資料進行增刪改查，也就是所謂的 CRUD 操作。

查詢
Query
Optimize the slow SQL query to reduce server load.
最佳化這個緩慢的 SQL 查詢以減少伺服器負載。

插入
Insert
Insert the new transaction data into the database safely.
將新交易資料安全地插入資料庫。

更新
Update
Update the order status to 'Shipped' when it leaves the warehouse.
當貨物離開倉庫時，將訂單狀態更新為「已出貨」。

刪除
Delete
Delete rows that are older than one year to save space.
刪除超過一年的資料列以節省空間。

截斷
Truncate
Truncate table removes all rows quickly but cannot be rolled back easily.
Truncate 快速移除所有資料列，但通常無法輕易回滾。

面對複雜的業務邏輯，我們需要以下更進階的查詢技巧與交易控制機制。

連接
Join
Use an inner join to combine data from the User and Order tables.
使用 Inner Join 來結合使用者和訂單資料表的資料。

聯集
Union
Use Union to combine result sets from two different tables.
使用 Union 來結合兩個不同資料表的結果集。

交易
Transaction
Wrap these operations in a transaction to ensure data integrity.
將這些操作包在一個交易中以確保資料完整性。

提交
Commit
Commit the transaction only if all steps are successful.
只有在所有步驟都成功時，才提交交易。

回滾
Rollback
Rollback the transaction immediately if an error occurs.
如果發生錯誤，立即回滾交易。

最後，為了確保系統穩定，我們需要了解這些資料庫管理與維護的功能。

檢視表
View
Create a view to simplify complex queries for the reporting team.
建立一個檢視表來為報表團隊簡化複雜查詢。

預存程序
Stored Procedure
Execute the stored procedure to generate the monthly report.
執行預存程序來產生月報表。

觸發器
Trigger
A trigger automatically updates the audit log when data changes.
當資料變更時，觸發器會自動更新稽核日誌。

正規化
Normalization
Normalization reduces data redundancy by organizing tables.
正規化透過組織資料表來減少資料冗餘。

遷移
Migration
Apply database migrations to update the schema to the latest version.
套用資料庫遷移以將綱要更新到最新版本。

連線池
Connection Pooling
Connection pooling improves performance by reusing active connections.
連線池透過重複使用活躍連線來提升效能。

備份
Backup
Schedule a daily backup to prevent data loss.
排程每日備份以防止資料遺失。

還原
Restore
Test the restore process regularly to ensure backups are valid.
定期測試還原流程以確保備份是有效的。


SQL 是資料操作的靈魂。熟練這些基本指令與維運概念，能讓你無論面對哪種關聯式資料庫都能得心應手。感謝收聽。
