歡迎來到 Web 開發基礎篇。無論你是前端還是後端，弄懂 HTTP 協定與 API 溝通方式，是所有 Web 開發者的共同語言。讓我們從最基礎的請求與回應開始。

我們從一次完整的 HTTP 互動開始，包含請求與回應。

請求
Request
The client sends an HTTP request to the server for data.
客戶端發送一個 HTTP 請求到伺服器獲取資料。

回應
Response
The server returns a JSON response containing the user profile.
伺服器回傳一個包含使用者資料的 JSON 回應。

要告訴伺服器做什麼動作，我們需要使用正確的 HTTP 方法與參數設定。

方法
Method
GET and POST are the most common HTTP methods used in APIs.
GET 和 POST 是 API 中最常用的 HTTP 方法。

端點
Endpoint
Hit the login endpoint to authenticate the user credentials.
呼叫登入端點以驗證使用者憑證。

參數
Parameter
Validate the input parameters strictly to avoid errors.
嚴格驗證輸入參數以避免錯誤。

查詢字串
Query String
Pass the search filters in the URL query string.
在 URL 查詢字串中傳遞搜尋篩選條件。

GraphQL
GraphQL is a query language for APIs that allows clients to request exactly what they need.
GraphQL 是一種 API 查詢語言，允許客戶端只請求他們需要的資料。

HTTP 訊息的內部結構，藏著許多重要資訊，通常分為標頭與主體。

標頭
Header
Authentication tokens are usually sent in the Authorization header.
驗證 Token 通常放在 Authorization 標頭中傳送。

主體
Body
The POST request body contains the form data submitted by the user.
POST 請求主體包含使用者提交的表單資料。

負載
Payload
The JSON payload size is too large for the server to process.
JSON 負載大小過大，伺服器無法處理。

內容類型
Content-Type
Set the Content-Type header to application/json.
將 Content-Type 標頭設為 application/json。

伺服器透過狀態碼來回應請求的結果，常見的類別如下，請務必熟悉。

未授權
Unauthorized
Return 401 Unauthorized if the user is not logged in.
如果使用者未登入，回傳 401 未授權。

禁止的
Forbidden
Return 403 Forbidden if the user lacks the necessary permission.
如果使用者缺乏必要權限，回傳 403 禁止的。

找不到
Not Found
The API endpoint returned a 404 Not Found error.
API 端點回傳 404 找不到錯誤。

內部伺服器錯誤
Internal Server Error
Check the server logs to investigate the 500 Internal Server Error.
檢查伺服器日誌以調查 500 內部伺服器錯誤。

最後，我們來看看網頁安全與狀態管理的機制，這些是保護使用者資料的關鍵。

跨來源資源共用
CORS
CORS 是 Cross-Origin Resource Sharing 的縮寫
Configure CORS to allow requests from the frontend domain.
設定 CORS 以允許來自前端網域的請求。

JWT
JWT 是 JSON Web Token 的縮寫。
JWT is commonly used for stateless authentication in mobile apps.
JWT 常被用於行動應用程式的無狀態驗證。

Cookie
Cookie
Secure cookies should have the HttpOnly flag to prevent XSS.
安全 Cookie 應該要有 HttpOnly 標記以防止 XSS。

工作階段
Session
The server stores the user state in a session on the backend.
伺服器將使用者狀態儲存在後端的 Session 中。

快取控制
Cache-Control
Use headers to define the cache-control policy for browsers.
使用標頭來定義瀏覽器的快取控制策略。


HTTP 是網際網路溝通的基礎。掌握這些 Request 與 Response 的細節，能幫助你更快地除錯與設計 API。希望這集的內容對你有所幫助，我們下次見。
