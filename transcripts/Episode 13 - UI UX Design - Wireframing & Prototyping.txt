歡迎來到 UI/UX 設計與測試篇。雖然工程師不一定會畫圖，但理解設計師的語言能讓合作更順暢。這集我們介紹從構想到原型的設計流程術語。

設計流程通常從發想與定義使用者開始，我們會用到以下核心概念。

使用者介面
UI
UI 是 User Interface 的縮寫
The UI should be intuitive and easy to navigate.
使用者介面應該直覺且易於瀏覽。

使用者體驗
UX
UX 是 User Experience 的縮寫
Good UX design focuses on the user's journey and satisfaction.
好的 UX 設計專注於使用者旅程與滿意度。

使用者旅程
User Journey
Map out the user journey to identify pain points.
繪製使用者旅程以找出痛點。

人物誌
Persona
Create a user persona to represent our target audience.
建立一個使用者人物誌來代表我們的目標受眾。

從概念到成品，需要經歷不同精細度的設計階段。

線框圖
Wireframe
A wireframe is a low-fidelity sketch of the layout.
線框圖是版面配置的低保真草圖。

原型
Prototype
We built an interactive prototype to test the user flow.
我們建立了一個互動式原型來測試使用者流程。

視覺稿
Mockup
The mockup shows the final colors and typography.
視覺稿展示了最終的顏色與字體排印。

Figma
Figma is a collaborative design tool used by our team.
Figma 是我們團隊使用的協作設計工具。

為了確保介面的美觀與易用性，設計系統是必不可少的，包含以下元素。

排版
Typography
Consistent typography improves readability and brand identity.
一致的排版提升了易讀性與品牌識別。

色票
Color Palette
Stick to the defined color palette for all UI elements.
所有 UI 元素都要堅守定義好的色票。

網格系統
Grid System
Use a 12-column grid system for alignment and consistency.
使用 12 欄網格系統來保持對齊與一致性。

空白
Whitespace
Use whitespace effectively to reduce visual clutter.
有效使用空白以減少視覺混亂。

響應式設計
Responsive Design
Responsive design adapts the layout to different screen sizes.
響應式設計使版面配置適應不同的螢幕尺寸。

我們來認識一些常見的 UI 元件，這些是構成畫面的基本單位。

導覽列
Navbar
The navbar contains links to the main sections of the site.
導覽列包含通往網站主要區塊的連結。

側邊欄
Sidebar
The sidebar provides quick access to filters and settings.
側邊欄提供快速存取篩選器與設定的功能。

模態視窗
Modal
Open a modal window to confirm the delete action.
開啟模態視窗以確認刪除動作。

輪播
Carousel
The carousel displays a slideshow of featured products.
輪播展示精選產品的幻燈片。

麵包屑
Breadcrumb
Breadcrumbs help users understand their location in the site hierarchy.
麵包屑協助使用者了解他們在網站階層中的位置。

行動呼籲
CTA
CTA 是 Call to Action 的縮寫
Example: The "Sign Up" button is the primary CTA on the landing page.
「註冊」按鈕是著陸頁上主要的行動呼籲。

最後，透過測試與迭代來驗證設計的成效，我們需要關注這些指標。

互動
Interaction
Micro-interactions make the app feel more alive.
微互動讓應用程式感覺更生動。

易用性
Usability
Conduct usability testing to fix interface issues.
進行易用性測試以修復介面問題。

A/B 測試
A/B Testing
We are running an A/B test to see which button performs better.
我們正在進行 A/B 測試，看看哪個按鈕表現更好。

無障礙
Accessibility
Accessibility ensures people with disabilities can use the app.
無障礙設計確保身心障礙人士也能使用這個應用程式。

風格指南
Style Guide
Follow the style guide to maintain visual consistency.
遵循風格指南以維持視覺一致性。


好的設計不僅是好看，更是好用。學會這些術語，你就能在設計評審中精確表達你的技術觀點。感謝收聽。
