歡迎來到軟體品質保證篇。寫出能夠運作的程式碼只是及格，寫出經過測試、穩固的程式碼才是專業。這集我們深入探討測試種類與 TDD 開發流程。

確保軟體品質，我們需要從不同層級進行測試，以下是常見的測試類型。

單元測試
Unit Testing
Unit testing verifies individual functions in isolation.
單元測試在隔離狀態下驗證個別函式。

整合測試
Integration Testing
Integration testing checks if different modules work together correctly.
整合測試檢查不同模組是否能正確協同運作。

端對端測試
E2E Testing
E2E Testing 是 End-to-End Testing 的縮寫
E2E testing simulates the real user scenario from start to finish.
端對端測試從頭到尾模擬真實使用者場景。

手動測試
Manual Testing
Manual testing is still necessary for UI and UX verification.
手動測試對於 UI 和 UX 驗證仍然是必要的。

冒煙測試
Smoke Test
Run a quick smoke test to verify the critical path is working.
執行快速的冒煙測試以驗證關鍵路徑運作正常。

回歸測試
Regression Testing
Run regression testing to ensure new changes didn't break existing features.
執行回歸測試以確保新變更沒有破壞現有功能。

負載測試
Load Testing
Load testing determines how the system behaves under high traffic.
負載測試決定系統在高流量下的行為表現。

壓力測試
Stress Testing
Stress testing pushes the system beyond its limits to see how it breaks.
壓力測試將系統推向極限，看看它如何崩潰。

不同的開發方法論，會影響我們撰寫測試的時機與方式，主要的流派如下。

測試驅動開發
TDD
TDD 是 Test-Driven Development 的縮寫
TDD requires writing tests before writing code.
測試驅動開發要求在寫程式碼前先寫測試。

行為驅動開發
BDD
BDD 是 Behavior-Driven Development 的縮寫
BDD uses human-readable descriptions for test cases.
行為驅動開發使用人類可讀的描述來撰寫測試案例。

黑箱測試
Black Box Testing
In black box testing, the tester doesn't look at the internal code.
在黑箱測試中，測試者不看內部程式碼。

白箱測試
White Box Testing
White box testing inspects the internal structures of the application.
白箱測試檢查應用程式的內部結構。

持續測試
Continuous Testing
Continuous testing executes automated tests as part of the CI pipeline.
持續測試作為 CI 管線的一部分執行自動化測試。

在撰寫測試程式碼時，我們會用到這些具體的技巧與指標。

測試案例
Test Case
Write a test case for every edge case you can think of.
為你能想到的每個邊界情況寫一個測試案例。

測試套件
Test Suite
The entire test suite takes 10 minutes to run.
整個測試套件需要 10 分鐘才能執行完畢。

斷言
Assertion
The assertion fails if the expected value does not match the actual value.
如果預期值與實際值不符，斷言就會失敗。

程式碼覆蓋率
Code Coverage
We aim for 80% code coverage on the backend project.
我們的目標是後端專案達到 80% 的程式碼覆蓋率。

模擬
Mocking
Mocking external APIs allows us to test without internet access.
模擬外部 API 讓我們可以在沒有網路的情況下進行測試。

存根
Stubbing
Use stubbing to provide fixed responses for function calls.
使用存根為函式呼叫提供固定的回應。

不穩定的測試
Flaky Test
A flaky test passes sometimes and fails others without code changes.
不穩定的測試有時通過有時失敗，即便程式碼沒有變動。

最後是關於品質管理與缺陷追蹤的術語，這些是 QA 日常工作的一部分。

品保
QA
QA 是 Quality Assurance 的縮寫
QA engineers are responsible for the overall quality of the release.
QA 工程師負責發布版本的整體品質。

驗收
Acceptance
User Acceptance Testing (UAT) is the final step before go-live.
使用者驗收測試 (UAT) 是上線前的最後一步。

缺陷
Defect
Log the defect in the bug tracking system with reproduction steps.
在 Bug 追蹤系統中記錄缺陷及其重現步驟。

修復
Fix
Deploy the hotfix to fix the critical bug immediately.
部署 Hotfix 以立即修復這個嚴重 Bug。


測試不是拖慢開發的累贅，而是加速重構的保障。養成寫測試的習慣，你的程式碼品質將會大幅提升。感謝收聽。
