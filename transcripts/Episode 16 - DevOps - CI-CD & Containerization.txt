歡迎來到 DevOps 與雲端原生篇。寫完程式只是第一步，如何將程式碼安全、自動化地部署到伺服器，是現代工程師的必修課。這集我們將重點放在 CI/CD 流程與容器化技術。

現代軟體交付依賴自動化的流程，從提交程式碼到上線一氣呵成，我們來看看 CI/CD 的各個環節。

管線
Pipeline
The build pipeline failed because a unit test didn't pass.
建置管線失敗了，因為有一個單元測試沒通過。

持續整合
CI
CI 是 Continuous Integration 的縮寫
Continuous Integration helps the team detect integration errors early.
持續整合有助於團隊早期發現整合錯誤。

持續交付
CD
CD 是 Continuous Delivery 的縮寫
Continuous Delivery automates the release process up to the approval stage.
持續交付自動化了直到核准前的發布流程。

持續部署
Continuous Deployment
Continuous Deployment releases updates automatically without human intervention.
持續部署在無人介入的情況下自動發布更新。

產物
Artifact
Store the build artifact in a secure repository for deployment.
將建置產物儲存在安全的檔案庫中以供部署。

環境
Environment
Do not test experimental features directly in the production environment.
不要直接在正式環境中測試實驗性功能。

預備環境
Staging
Deploy the code to staging for final verification before live release.
在正式發布前，部署程式碼到預備環境進行最終驗證。

正式環境
Production
A critical bug was found in production affecting real users.
在正式環境發現了一個影響真實使用者的嚴重 Bug。

管理基礎設施不再是手動操作，而是程式碼化與可觀察化，相關技術如下。

基礎設施即程式碼
Infrastructure as Code
Use Terraform to manage your Infrastructure as Code.
使用 Terraform 來管理你的基礎設施即程式碼。

供應
Provision
The script automatically provisions a new virtual machine.
這個腳本會自動供應一台新的虛擬機器。

監控
Monitoring
Monitoring tells us the real-time health of the system.
監控告訴我們系統的即時健康狀態。

日誌
Logging
Check the centralized error logs to diagnose the issue.
檢查集中式錯誤日誌以診斷問題。

指標
Metric
CPU usage is a key metric for scaling decisions.
CPU 使用率是決定擴展與否的關鍵指標。

警報
Alert
Set up an alert to notify the team when memory consumption is high.
設定警報，在記憶體消耗過高時通知團隊。

容器化技術改變了我們打包與部署應用程式的方式，以下是 Docker 的核心概念。

容器
Container
Containers package code and dependencies together for consistency.
容器將程式碼與相依性打包在一起以確保一致性。

映像檔
Image
Build a Docker image from the Dockerfile instructions.
根據 Dockerfile 的指令建置 Docker 映像檔。

註冊表
Registry
Push the built image to the private registry securely.
將建置好的映像檔安全地推送到私有註冊表。

編排
Orchestration
Kubernetes is the standard platform for container orchestration.
Kubernetes 是容器編排的標準平台。

Kubernetes 是容器調度的標準，雖然複雜但功能強大，這些是 K8s 的基本組成。

叢集
Cluster
The cluster consists of multiple nodes working together.
這個叢集由多個協同工作的節點組成。

節點
Node
A node can be a physical server or a virtual machine in the cluster.
節點可以是叢集中的實體伺服器或虛擬機器。

Pod
A pod is the smallest deployable unit in Kubernetes.
Pod 是 Kubernetes 中最小的可部署單位。

服務
Service
Create a service to expose the pods to the network.
建立一個 Service 將 Pod 暴露給網路。

副本
Replica
Set the replica count to 3 to ensure high availability.
將副本數設為 3 以確保高可用性。

入口
Ingress
Configure ingress rules to route external traffic to internal services.
設定 Ingress 規則以將外部流量路由至內部服務。

機密資訊
Secret
Store sensitive API keys in K8s secrets, not in plain text.
將敏感的 API Key 儲存在 K8s Secrets 中，而非明文儲存。


CI/CD 與容器化已是業界標準。熟悉這些流程不僅能減少重複勞動，更能提升發布的品質與信心。感謝收聽。
