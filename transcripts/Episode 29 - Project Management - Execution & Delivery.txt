歡迎來到專案執行與交付篇。在規劃之後，就是緊湊的執行與交付。這集涵蓋了開發流程、測試驗收以及與 QA 協作時會用到的重要術語。

敏捷開發是目前業界的主流方法論，讓我們首先來認識它的核心詞彙。

敏捷
Agile
Agile methodology emphasizes iteration and flexibility.
敏捷方法論強調迭代與彈性。

衝刺
Sprint
We plan our tasks for the upcoming two-week sprint.
我們為接下來兩週的衝刺規劃任務。

待辦清單
Backlog
Prioritize items in the product backlog for the next release.
為下一次發布排定產品待辦事項的優先順序。

啟動會議
Kick-off
The project kick-off meeting is scheduled for tomorrow.
專案啟動會議排定在明天。

衝刺規劃
Sprint Planning
We estimate task complexity during sprint planning.
我們在衝刺規劃時估算任務複雜度。

每日站立會議
Daily Stand-up
I'll give my update at the daily stand-up.
我會在每日站立會議報告我的進度。

估算
Estimation
What is your time estimation for completing this feature?
你對完成這個功能的時間估算是多少？

完成
Done
Definition of Done means coded, tested, and documented.
「完成」的定義意指已編碼、已測試且已撰寫文件。

有效管理任務狀態能避免進度延遲，接下來讓我們來看看常見的看板狀態。

待辦事項
To-do
Add this bug fix to your to-do list for today.
把這個 Bug 修復加到你今天的待辦事項清單。

進行中
In Progress
The ticket status is currently In Progress.
這張票的狀態目前是進行中。

阻礙
Blocker
Please report any blockers immediately to the Scrum Master.
請立即向 Scrum Master 回報任何阻礙。

範圍蔓延
Scope Creep
We need to stick to the plan and prevent scope creep.
我們需要堅持計畫並防止範圍蔓延。

優先級
Priority
Set the priority to High for this critical bug.
將這個嚴重 Bug 的優先級設為高。

嚴重性
Severity
This is a high severity bug causing data loss.
這是一個導致資料遺失的高嚴重性 Bug。

品質保證與驗收是交付前的最後一哩路，接下來讓我們來看看測試階段的關鍵術語。

品質保證
QA
QA 是 Quality Assurance 的縮寫
QA found a bug during the regression test.
QA 在回歸測試期間發現了一個 Bug。

測試環境
Testbed
Run the automated script in the testbed first.
先在測試環境中執行自動化腳本。

使用者驗收測試
UAT
UAT 是 User Acceptance Testing 的縮寫
The client is performing UAT to sign off the release.
客戶正在進行 UAT 以簽核這次發布。

簽核
Sign-off
We need final sign-off from the client before launch.
發布前我們需要客戶的最終簽核。

修補程式
Hotfix
We need to release a hotfix immediately to patch the hole.
我們需要立即發布 Hotfix 來修補這個漏洞。

為了確保產品符合需求，接下來讓我們來關注以下使用者體驗相關的詞彙。

痛點
Pain Point
Our goal is to solve the user's biggest pain point.
我們的目標是解決使用者最大的痛點。

使用者體驗
UX
UX 是 User Experience 的縮寫
We need to improve the UX flow for registration.
我們需要改善註冊的 UX 流程。

使用者介面
UI
UI 是 User Interface 的縮寫
The UI needs a refresh to look more modern.
UI 需要更新以看起來更現代。

跨功能
Cross-functional
We work in a cross-functional team with designers and QA.
我們與設計師和 QA 在跨功能團隊中工作。

可行性
Feasibility
Please analyze the technical feasibility of this idea.
請分析這個點子的技術可行性。

基準
Benchmark
Use the current performance as a benchmark for improvement.
使用目前的效能作為改進的基準。


專案的成功往往取決於細節的執行。希望這些術語能幫助你在每個開發環節中都保持精確與專業。感謝收聽。
