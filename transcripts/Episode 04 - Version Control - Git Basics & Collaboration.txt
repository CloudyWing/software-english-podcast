歡迎來到版本控制基礎篇。Git 是現代開發者每天賴以生存的工具。我們將從基礎指令到團隊協作的術語一一介紹，幫助你建立穩固的開發習慣。

Git 的一切都從建立與管理檔案庫開始，我們首先熟悉這些初始化相關的術語。

版本控制
Version Control
Git is the most popular distributed version control system.
Git 是最受歡迎的分散式版本控制系統。

檔案庫
Repository
Please clone the repository to your local machine.
請將檔案庫複製到你的本機。

初始化
Initialize
Run git init to start a new project in this folder.
執行 git init 在這個資料夾開始一個新專案。

複製
Clone
Clone the project via the HTTPS URL provided on GitHub.
透過 GitHub 提供的 HTTPS 網址來複製專案。

工作目錄
Working Directory
Your changes are currently sitting in the working directory.
你的變更目前正位於工作目錄中。

暫存區
Staging Area
You must add files to the staging area before you can commit them.
在提交之前，你必須先將檔案加入暫存區。

加入
Add
Don't forget to add the new files to git tracking.
別忘了將新檔案加入 git 追蹤。

提交
Commit
Write a meaningful commit message explaining the change.
寫一個有意義的提交訊息來解釋這次變更。

狀態
Status
Check the git status to see which files have been modified.
檢查 git status 看看哪些檔案被修改了。

分支操作是同時進行多項開發任務的關鍵，讓我們看看如何管理與切換分支。

分支
Branch
Create a feature branch to work on the new task.
建立一個功能分支來進行這項新任務。

主分支
Main
The production code usually lives in the main branch.
正式環境的程式碼通常位於 main 分支。

簽出
Checkout
Checkout the main branch before you pull the latest updates.
在拉取最新更新前，先切換到 main 分支。

切換
Switch
Use git switch to change branches safely without confusion.
使用 git switch 來安全且清楚地切換分支。

合併
Merge
We need to merge the feature branch back into the develop branch.
我們需要將功能分支合併回開發分支。

重訂基底
Rebase
Rebase your branch to keep a clean and linear history.
對你的分支進行 Rebase 以保持乾淨且線性的歷史紀錄。

衝突
Conflict
You need to resolve the merge conflict manually in your editor.
你需要在編輯器中手動解決合併衝突。

解決
Resolve
Mark the file as resolved after you fix the code conflict.
修正程式碼衝突後，將檔案標記為已解決。

標籤
Tag
Create a version tag like v1.0 for this release.
為這次發布建立一個像 v1.0 的版本標籤。

透過遠端操作才能與團隊同步，以下是與伺服器互動的常用指令。

遠端
Remote
Add the GitHub repository as a remote origin.
將 GitHub 檔案庫新增為遠端 origin。

原始遠端
Origin
Push your local changes to origin.
將你的本機變更推送到 origin。

獲取
Fetch
Fetch the latest changes from the server without merging them.
從伺服器獲取最新變更，但不進行合併。

拉取
Pull
Pull origin main to update your local code immediately.
Pull origin main 來立即更新你的本機程式碼。

推送
Push
You cannot push directly to a protected branch.
你不能直接推送至受保護的分支。

拉取請求
Pull Request
Open a pull request so the team can review your code.
開啟一個 Pull Request，這樣團隊才能審查你的程式碼。


Git 的指令雖然多，但只要掌握這幾個最核心的動作，就能應對絕大多數的工作場景。建議您反覆聆聽，特別是那些容易混淆的觀念。感謝收聽。
