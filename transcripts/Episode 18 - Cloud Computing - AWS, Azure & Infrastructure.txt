歡迎來到雲端運算篇。這一集我們將焦點放在具體的雲端供應商與基礎設施環境。無論是 AWS 還是 Azure，這些都是你部署程式碼的地方。

三大雲端龍頭提供了各式各樣的服務，接下來我們從市佔率最高的這三家開始。

Azure
We deploy our .NET applications to Microsoft Azure App Service.
我們將 .NET 應用程式部署到 Microsoft Azure App Service。

AWS
AWS 是 Amazon Web Services 的縮寫
AWS offers a wide range of cloud infrastructure services.
AWS 提供廣泛的雲端基礎設施服務。

Google Cloud
We use Google Cloud Platform for its strong data analytics tools.
我們使用 Google Cloud Platform，因為它有強大的資料分析工具。

雲端服務主要分為三種模式，了解這些層級有助於你選擇正確的解決方案。

基礎設施即服務
IaaS
IaaS allows you to rent virtual servers and storage. IaaS stands for Infrastructure as a Service.
IaaS 讓你租用虛擬伺服器與儲存空間。

平台即服務
PaaS
PaaS provides a platform for developers to build and deploy apps without managing servers. PaaS stands for Platform as a Service.
PaaS 為開發者提供了一個平台來建構與部署應用程式，而無需管理伺服器。

軟體即服務
SaaS
SaaS delivers software applications over the internet. SaaS stands for Software as a Service.
SaaS 透過網際網路交付軟體應用程式。

在部署資源前，了解基礎網路架構是必須的，接下來這些術語定義了雲端的地理與邏輯邊界。

區域
Region
Select the region closest to your users to reduce latency.
選擇最接近你使用者的區域以減少延遲。

可用區域
Availability Zone
Deploy resources across multiple availability zones for redundancy.
跨多個可用區域部署資源以達冗餘效果。

虛擬私有雲
VPC
VPC 是 Virtual Private Cloud 的縮寫
Isolate your database in a secure VPC.
將你的資料庫隔離在安全的 VPC (虛擬私有雲) 中。

子網段
Subnet
Organize your network into public and private subnets.
將你的網路組織成公有和私有子網段。

身份存取管理
IAM
IAM 是 Identity and Access Management 的縮寫
Use IAM roles to manage permissions securely.
使用 IAM 角色來安全地管理權限。

運算資源是雲端轉型的核心，從虛擬機到無伺服器架構，選擇相當多元，讓我們接著往下看。

虛擬機器
Virtual Machine
Migrate the legacy app to a cloud-based virtual machine (VM).
將舊版應用程式遷移到雲端虛擬機器。

EC2
EC2 是 Elastic Compute Cloud 的縮寫
Launch an EC2 instance to run the backend server.
啟動一個 EC2 執行個體來執行後端伺服器。

無伺服器
Serverless
Azure Functions is a serverless solution for event-driven code.
Azure Functions 是一個針對事件驅動程式碼的無伺服器解決方案。

Lambda
AWS Lambda lets you run code without provisioning servers.
AWS Lambda 讓你不需供應伺服器即可執行程式碼。

冷啟動
Cold Start
Cold start latency is a common concern in serverless architectures.
冷啟動延遲是無伺服器架構中常見的隱憂。

儲存與靜態資源的配置也是一大學問，常見的服務類型包含接下來的這幾種。

S3
S3 是 Simple Storage Service 的縮寫
Store the user uploaded images in an Amazon S3 bucket.
將使用者上傳的圖片儲存在 Amazon S3 儲存桶中。

Blob 儲存
Blob Storage
Azure Blob Storage is used for storing massive amounts of unstructured data.
Azure Blob Storage 用於儲存大量的非結構化資料。

內容傳遞網路
CDN
CDN 是 Content Delivery Network 的縮寫
Use a CDN to speed up the delivery of static assets globally.
使用 CDN 加速全球靜態資源的傳遞。

最後，我們來看關於擴展性與進階架構的術語，這是雲端真正發揮威力的地方，讓我們接著往下看。

可擴展性
Scalability
Cloud platforms offer effortless horizontal scalability.
雲端平台提供輕鬆的水平擴展性。

自動擴展
Autoscaling
Enable autoscaling to handle sudden traffic spikes.
啟用自動擴展以處理突發的流量高峰。

負載平衡器
Load Balancer
The load balancer distributes incoming traffic across healthy servers.
負載平衡器將進入的流量分配到健康的伺服器上。

多區域
Multi-region
Deploy to a multi-region architecture for disaster recovery.
部署到多區域架構以進行災難復原。

多租戶
Multi-tenancy
The platform supports multi-tenancy for serving different clients securely.
這個平台支援多租戶，以安全地服務不同客戶。

容器註冊表
Container Registry
Push your Docker images to Azure Container Registry (ACR).
將你的 Docker 映像檔推送到 Azure Container Registry。


雲端平台賦予了開發者強大的基礎設施能力。不管你選擇哪一家供應商，理解這些核心組件將幫助你構建更穩固的系統。感謝收聽。
