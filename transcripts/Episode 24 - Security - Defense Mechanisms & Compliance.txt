歡迎來到資安防護與合規篇。了解了攻擊手法後，這集我們學習如何學習如何建立防線。從身分驗證、加密標準到合規規範，這些是構建安全系統的基石。

密碼學是保護資料機密性與完整性的核心技術，以下是幾個關鍵概念。

加密
Encryption
Encryption protects data confidentiality during transmission.
加密保護傳輸過程中資料的機密性。

解密
Decryption
You need a private key for the decryption of this file.
你需要私鑰來對這個檔案進行解密。

雜湊
Hash
Compare the file hash to verify it hasn't been altered.
比較檔案雜湊值以驗證它未被竄改。

加鹽
Salt
Always salt passwords before hashing to prevent rainbow table attacks.
在雜湊之前務必將密碼加鹽以防止彩虹表攻擊。

對稱式
Symmetric
Symmetric encryption uses the same key for both locking and unlocking.
對稱式加密使用相同的金鑰進行上鎖與解鎖。

非對稱式
Asymmetric
Asymmetric encryption uses a public and private key pair.
非對稱式加密使用一對公鑰和私鑰。

傳輸層安全性
TLS
TLS 是 Transport Layer Security 的縮寫
TLS encrypts traffic over the internet, replacing SSL.
TLS 加密網際網路上的流量，取代了 SSL。

嚴格的存取控制能有效防止未經授權的行為，接下來我們需要區分這幾個術語。

身分驗證
Authentication
Enable two-factor authentication to secure your account.
啟用雙因子身分驗證以保護你的帳號。

授權
Authorization
Authorization determines what resources a user can access.
授權決定了使用者可以存取哪些資源。

憑證
Credential
Do not hardcode credentials like passwords in the source code.
不要將密碼等憑證寫死在原始碼中。

權杖
Token
The access token expires in one hour, requiring re-login.
存取權杖在一小時後過期，需要重新登入。

特權
Privilege
Follow the principle of least privilege for user accounts.
對使用者帳號遵循最小特權原則。

網路安全設施是抵禦外部攻擊的第一道防線，常見的防護設備包含接下來這幾種。

防火牆
Firewall
Configure the firewall to block suspicious incoming traffic.
設定防火牆以封鎖可疑的傳入流量。

虛擬私人網路
VPN
VPN 是 Virtual Private Network 的縮寫
Use a VPN to access the internal corporate network securely.
使用 VPN 安全地存取內部企業網路。

零信任
Zero Trust
Zero Trust assumes no user or device is trusted by default, even inside the network.
零信任假設沒有使用者或裝置是預設可信的，即便是網路內部也一樣。

隔離區
DMZ
DMZ 是 Demilitarized Zone 的縮寫
Place public-facing servers in the DMZ for added security.
將面對公眾的伺服器放置在 DMZ 以增加安全性。

沙盒
Sandbox
Run untrusted code in a sandbox to isolate it from the system.
在沙盒中執行不受信任的程式碼以將其與系統隔離。

蜜罐
Honeypot
The honeypot attracts attackers so we can analyze their methods.
蜜罐吸引攻擊者以便我們分析他們的手法。

最後，透過流程管理與測試來確保整體的安全性，接下來這些是合規的必要步驟。

CIA 三角
CIA Triad
CIA 是 Confidentiality, Integrity, and Availability 的縮寫
Security is often defined by the CIA triad: Confidentiality, Integrity, and Availability.
安全性通常由 CIA 三角來定義：機密性、完整性與可用性。

合規
Compliance
We must ensure compliance with GDPR regulations.
我們必須確保符合 GDPR 法規。

風險
Risk
Conduct a risk assessment before deploying new features.
部署新功能前進行風險評估。

減緩
Mitigation
Implement mitigation strategies to reduce the impact of threats.
實作減緩策略以降低威脅的影響。

事件
Incident
Report the security incident to the response team immediately.
立即向應變團隊通報資安事件。

滲透測試
Penetration Test
We perform a penetration test annually to find security holes.
我們每年執行一次滲透測試以找出安全漏洞。

消毒
Sanitize
Sanitize user inputs to prevent malicious scripts from running.
消毒使用者輸入以防止惡意腳本執行。


資訊安全是不容忽視的領域，這裡提到的每個術語都可能關係到公司的生死存亡。請務必熟悉它們。感謝收聽。
